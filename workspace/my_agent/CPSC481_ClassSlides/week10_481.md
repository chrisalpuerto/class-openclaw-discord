# CPSC 481 Week 10: Neural Networks as Function Approximators

## The Shift to Deep Learning
* **Limitations of Previous Models**: Linear and logistic regression struggle with high-dimensional data like images. Decision trees become unmanageably deep for pixel-level tasks.
* **Core Goal**: Transition from hand-engineered features to **Representation Learning**, where the network learns which features are important.
* **Definition of "Deep"**: Organized into many hidden layers between input and output to compute new representations.
* **Universal Approximation Theorem**: A single hidden layer can approximate any continuous function, but deep networks are exponentially more efficient for complex tasks.

## Neural Network Components
* **The Perceptron**: A simple unit where computation is $z = \sum (w_i x_i) + b$ and activation is $a = g(z)$. Weights and bias are the tunable parameters.
* **Activation Functions (Non-Linearity)**: Necessary because without them, multiple layers collapse into a single linear layer.
    * **Sigmoid**: Squashes output to [0,1].
    * **ReLU (Rectified Linear Unit)**: Most popular for deep nets; computationally cheap and solves the vanishing gradient problem.
    * **Tanh**: Squashes output to [-1,1].
* **Loss Functions**: Measured by Mean Squared Error (MSE) for regression or Cross-Entropy for classification.

## Training Mechanisms
* **Backpropagation**: The fundamental algorithm using the Calculus Chain Rule to distribute "blame" (gradients) from output back to input.
* **Optimizers**: Weights are updated using Gradient Descent.
    * **Mini-batch SGD**: Provides the best balance of speed and stability.
    * **Adam**: Most popular modern optimizer; automatically adjusts learning rates for each weight.
* **Regularization**: Techniques like **Dropout** (randomly killing neurons) or **L2 Weight Decay** (penalizing large weights) are used to prevent overfitting.

## Implementation & Hardware
* **Linear Algebra**: Math is handled via matrix multiplication ($Y = XW + b$), which is optimized for GPUs and TPUs.
* **Frameworks**: PyTorch is favored by researchers for dynamic graphs, while TensorFlow/Keras is an industry standard for static graphs.
