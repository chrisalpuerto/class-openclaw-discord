# CPSC 481 Week 6: Heuristic Search Strategies

## From Blind to Informed Search
* **Limitation**: Uninformed searches like BFS and DFS expand in all directions, leading to exponential complexity $O(b^d)$ and memory bottlenecks.
* **Informed Search**: Uses an **Evaluation Function** $f(n)$ to select nodes based on domain knowledge.
* **Heuristic Function $h(n)$**: Estimates the cost from node $n$ to the goal. A common example is straight-line distance.

## A* Search
* **Core Function**: $f(n) = g(n) + h(n)$.
* **$g(n)$**: Cost to reach the node.
* **$h(n)$**: Estimated cost to the goal.
* **Admissibility**: A heuristic is admissible if it never overestimates the true cost to the goal.
* **Consistency (Monotonicity)**: A heuristic is consistent if for every node $n$ and successor $n'$, $h(n) \le c(n, a, n') + h(n')$.

## Search Comparison
* **Greedy Best-First Search**: Expands the node that appears closest to the goal ($f(n) = h(n)$). It is not optimal and can be trapped in loops.
* **A* Optimality**: A* is complete and optimal if the heuristic is admissible (for tree search) or consistent (for graph search).
