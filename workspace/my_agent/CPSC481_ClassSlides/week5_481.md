# CPSC 481 Week 5: Solving Problems by Searching

## Problem-Solving Agents
* **Paradigm**: **Plan-then-Execute** (open-loop). The agent looks ahead in its "head" before committing to an action.
* **Atomic Representation**: States are treated as "black boxes" (e.g., city names like "Arad") with no visible internal structure.
* **Search Tree**: Explores paths from the root (initial state). A state in the graph may appear as many nodes in the tree.

## Components of a Well-Defined Problem
1. **Initial State**: Where the agent begins.
2. **Actions(s)**: The set of possible actions from state *s*.
3. **Transition Model**: Result(*s, a*) returns the outcome state.
4. **Goal Test**: A Boolean function determines if a state is the target (**Explicit** specific state or **Implicit** property).
5. **Path Cost**: A function (*g*) representing the total cost (time, fuel, money) to reach a state.

## Search Performance Metrics
* **Completeness**: Finds a solution if one exists.
* **Optimality**: Finds the lowest-cost solution.
* **Complexity**: Measured by **b** (branching factor), **d** (depth of shallowest goal), and **m** (max path length).

## Uninformed (Blind) Search Strategies
* **Breadth-First Search (BFS)**: Expands shallowest node first (FIFO queue). Optimal if all step costs are equal. High memory usage: $O(b^d)$.
* **Uniform-Cost Search (Dijkstra)**: Expands node with lowest path cost *g(n)* (Priority Queue). Optimal for any non-negative cost.
* **Depth-First Search (DFS)**: Expands deepest node first (LIFO stack). Memory efficient $O(bm)$ but not complete or optimal.
* **Iterative Deepening Search (IDS)**: Repeatedly applies DFS with increasing depth limits. The best overall choice for large state spaces, combining BFS and DFS benefits.
* **Bidirectional Search**: Searches forward from start and backward from goal simultaneously. Complexity $O(b^{d/2})$.

## Informed (Heuristic) Search
* **Heuristic Function $h(n)$**: An estimated cost from node *n* to the goal (e.g., straight-line distance).
* **Goal**: Expand nodes that appear to be closer to the goal first to improve efficiency.
